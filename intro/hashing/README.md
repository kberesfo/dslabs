# Implementing a Hash Table with Separate Chaining

## Problem Overview

You will implement a `HashTable` class that stores:

- **string keys**
- **int values**

You are NOT allowed to use:

- `std::unordered_map`
- `std::map`

You must implement the data structure manually.

## Part 2 -- Hash Function

Implement a polynomial rolling hash:

```cpp
int HashTable::hashFunction(const string& key) const {
    const int prime = 31;
    long long hash = 0;

    for (char c : key) {
        hash = hash * prime + c;
    }

    return hash % capacity;
}
```

Implementation can be found in [hash_table.cpp](./src/hash_table.cpp)

### Note

I updated the hashing function to prevent long long overflow.

## Part 3 -- Insert & Collision Handling

For collision handling we are implementing a chaining mechanism. So for each key that collides we create a doubly linked list node that points to the next node. So for retrieval we iterate over the bucket and search for the matching key.

If our hash idx returns a non-empty bucket we iterate over if we find a matching key we update the value. Otherwise if we are inserting a node into a non-empty bucket we increment the private attribute `collisionCount`.

Implementation can be found in [hash_table.cpp](./src/hash_table.cpp)

## Theoretical Collision Analysis

Under the simple uniform hashing assumption, each key is equally likely to hash to any of the `m` buckets, and each hash is independent.

---

### Method 1 — Expected Number of Colliding Pairs

For two distinct keys $ k_1 $ and $ k_2 $:

$$\Pr[h(k_1) = h(k_2)] = \frac{1}{m}$$

If we insert $ n $ keys, the number of distinct unordered key pairs is:

$$\binom{n}{2} = \frac{n(n-1)}{2}$$

Since each pair collides with probability $ 1/m $, the expected number of colliding pairs is:

$$\mathbb{E}[\text{colliding pairs}] = \frac{n(n-1)}{2m}$$

For this experiment:

- $ n = 100 $
- $ m = 176 $

$$\mathbb{E}[\text{collisions}] = \frac{100(99)}{2 \cdot 176} = \frac{9900}{352}\approx 28.1$$

So we expect approximately **28 colliding pairs** under uniform hashing.

Note: This counts _all pairwise collisions_. If a bucket contains 3 elements, that contributes 3 colliding pairs.

From the printed hash table:

- 13 buckets contain **2 elements** → $(13 \times 1 = 13)$ pair collisions
- 5 buckets contain **3 elements** → $(5 \times 3 = 15)$ pair collisions
- Total pair collisions: $13 + 15 = 28$

The total pair collisions this measured value lines up nicely with our **theoretical value** $28.1$

---

### Method 2 — Expected Number of Insert Collisions (Bucket Occupancy Model)

Another way to analyze collisions is by computing the expected number of non-empty buckets.

Under the assumption of **uniform hashing**, the probability that a key hashes to a specific bucket is

$$\frac{1}{m}$$

Therefore, the probability that a key **does not** hash to that bucket is

$$1 - \frac{1}{m}$$

If \(n\) keys are inserted, the probability that **none of the keys** land in a particular bucket is

$$\left(1 - \frac{1}{m}\right)^n$$

Let $(X_i)$ be an indicator variable defined as

$$X_i =\begin{cases}1 & \text{if bucket } i \text{ is empty} \\ 0 & \text{otherwise}\end{cases}$$

Then the expected value of \(X_i\) is the probability that bucket \(i\) is empty:

$$\mathbb{E}[X_i] = \left(1 - \frac{1}{m}\right)^n$$

Since there are $(m)$ buckets, the expected number of empty buckets is

$$\mathbb{E}[empty] = \mathbb{E}[X] = m\left(1 - \frac{1}{m}\right)^n$$

The expected number of **non-empty buckets** is therefore

$$\mathbb{E}[\text{non-empty}] =m - m\left(1 - \frac{1}{m}\right)^n$$

which simplifies to

$$\mathbb{E}[\text{non-empty}] = m\left(1 - \left(1 - \frac{1}{m}\right)^n\right)$$

---

### Applying the Formula

For this experiment:

- $m = 176$ buckets
- $n = 100$ keys

$$176 \left(1 - \left(1 - \frac{1}{176}\right)^{100}\right)$$

$$\left(1 - \frac{1}{176}\right)^{100} \approx 0.5656$$

So the expected number of non-empty buckets is

$$176(1 - 0.566) \approx 176(0.434) \approx 76.4$$

---

### Expected Insert Collisions

Insert collisions occur whenever a key is inserted into a bucket that already contains a key.  
Since the first key placed in each bucket does not cause a collision, the expected number of insert collisions is

$$\text{Insert Collisions} = n - (\text{non-empty buckets})$$

Substituting our values:

$$100 - 76.4 \approx 23.6$$

The observed number of collisions in the experiment was **23**, which closely matches the theoretical expectation.

---

## Interpretation

The observed collision count aligns closely with theoretical expectations under uniform hashing. The random key set behaves approximately as predicted by probability theory, confirming that the hash function distributes keys reasonably uniformly across buckets.
